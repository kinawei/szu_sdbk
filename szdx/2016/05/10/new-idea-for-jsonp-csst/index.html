JSONP新思考：CSST

  • csst
  • js
  • jsonp
前端

前天早上在逛尤大的微博（@尤小右）的时候，看到他转了一条微博，关于 csst，第一次看到这个词，好奇的点进了 Github 的链接。实践了一把，由此有了这篇博文。

csst 作者的 Github 链接：zswang/csst

引言

csst 顾名思义 CSS Text Transformation 是一种类似于 jsonp 的跨域传输方案，不同于 jsonp 使用 <script> 突破浏览器域名限制，csst 使用的是 <link> 标签来完成这项工作。

本文将结合 csst Github 的讲解和我自己的实践，实实在在的封装一个 csst 模块。

原理

根据作者 Github 的讲解，csst 采用 <link> 标签进行跨域操作，同 jsonp 一样，只支持以 GET 方式调用 api 。通过后端封装 content-type 为 text/css 文件，给前端调用，达到跨域的效果。

后端将要传输的数据编译成 json string 然后 base64 编码后放入指定的 content 中，前端接收到css，浏览器会渲染读取到的css样式，监听 animationstart 和 webkitAnimationStart 两个事件，读取并反向解码最终即可得到数据。

优点

安全性佳：无需考虑传输过来的数据含有攻击性脚本，所有的数据都需要解析之后才能运行，而 jsonp 直接将数据读取到 <script> 中可能带来安全问题。

缺点

后端的模板比 jsonp 繁琐
